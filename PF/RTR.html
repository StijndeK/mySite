Real-Time Risers

A reactive game audio solution

Adapting parameters of audio in real-time, instead of playing cutup pieces of the audio, improves the reactivity of the audio and allows for the implementation of more dynamic sounds such as risers. RTR is a prototype for an adaptive audio tool that lets the user design a riser that adapts to game data in real-time. RTR has been developed for a my master's research into adressing certain issues in the common game audio development pipeline.

Pipeline

A quick and easy workflow allows the user to make changes to the riser and immediatly hear the riser back in game. The tool is designed similarly to standard audio middleware, using a seperate studio solution and engine integration plugin that are build on the same low-level audio system. This way the tool can be easily integrated in any engine and is not restricted in any way.

Risers

Risers are efficient low-cost audio effects. However, because risers are synchronised to their start as well as their point of impact, they are difficult to implement in an adaptive system. Risers generally consist of three dynamic elements: the attack, the impact, and the release. By modulating the audio data in between the start and impact, the riser can go on for a longer or shorter amount of time.

Integration

Implement the riser fast and efficiently. An Unreal Engine 4 plugin with blueprint nodes and actors is provided. The actors serve as the start and stop location of the riser and can be placed anywhere in the level. This way implementing the riser will only take a few seconds. Alternatively, blueprints can be used, for example when the start and/or stop position are not physical locations.
